310 ## Version 9.0.0
311 
312 The new major version brings a reworked styling system. Highlight.js now defines
313 a limited set of highlightable classes giving a consistent result across all the
314 styles and languages. You can read a more detailed explanation and background in
315 the [tracking issue][#348] that started this long process back in May.
316 
317 This change is backwards incompatible for those who uses highlight.js with a
318 custom stylesheet. The [new style guide][sg] explains how to write styles
319 in this new world.
320 
321 Bundled themes have also suffered a significant amount of improvements and may
322 look different in places, but all the things now consistent and make more sense.
323 Among others, the Default style has got a refresh and will probably be tweaked
324 some more in next releases. Please do give your feedback in our
325 [issue tracker][issues].

564 apart from that it's got smarter: by presenting languages in groups it avoids
565 running 10000 highlighting attempts after first load which was slowing it down
566 and giving bad overall impression. It is now also being generated from test
567 code snippets so the authors of new languages don't have to update both tests
568 and the demo page with the same thing.

577  target for that anymore.

579 - Clojure's data structures and literals are now highlighted outside of lists
580  and we can now highlight Clojure's REPL sessions.

781 ## Version 8.0
782 
783 This new major release is quite a big overhaul bringing both new features and
784 some backwards incompatible changes. However, chances are that the majority of
785 users won't be affected by the latter: the basic scenario described in the
786 README is left intact.
787 
788 Here's what did change in an incompatible way:
789 
790 - We're now prefixing all classes located in [CSS classes reference][cr] with
791  `hljs-`, by default, because some class names would collide with other
792  people's stylesheets. If you were using an older version, you might still want
793  the previous behavior, but still want to upgrade. To suppress this new
794  behavior, you would initialize like so:
795 
796  ```html
797  <script type="text/javascript">
798  hljs.configure({classPrefix: ''});
799  hljs.initHighlightingOnLoad();
800  </script>
801  ```
802 
803 - `tabReplace` and `useBR` that were used in different places are also unified
804  into the global options object and are to be set using `configure(options)`.
805  This function is documented in our [API docs][]. Also note that these
806  parameters are gone from `highlightBlock` and `fixMarkup` which are now also
807  rely on `configure`.
808 
809 - We removed public-facing (though undocumented) object `hljs.LANGUAGES` which
810  was used to register languages with the library in favor of two new methods:
811  `registerLanguage` and `getLanguage`. Both are documented in our [API docs][].
812 
813 - Result returned from `highlight` and `highlightAuto` no longer contains two
814  separate attributes contributing to relevance score, `relevance` and
815  `keyword_count`. They are now unified in `relevance`.
816 
817 Another technically compatible change that nonetheless might need attention:
818 
819 - The structure of the NPM package was refactored, so if you had installed it
820  locally, you'll have to update your paths. The usual `require('highlight.js')`
821  works as before. This is contributed by [Dmitry Smolin][].
822 
823 New features:
824 
825 - Languages now can be recognized by multiple names like "js" for JavaScript or
826  "html" for, well, HTML (which earlier insisted on calling it "xml"). These
827  aliases can be specified in the class attribute of the code container in your
828  HTML as well as in various API calls. For now there are only a few very common
829  aliases but we'll expand it in the future. All of them are listed in the
830  [class reference][cr].
831 
832 - Language detection can now be restricted to a subset of languages relevant in
833  a given context — a web page or even a single highlighting call. This is
834  especially useful for node.js build that includes all the known languages.
835  Another example is a StackOverflow-style site where users specify languages
836  as tags rather than in the markdown-formatted code snippets. This is
837  documented in the [API reference][] (see methods `highlightAuto` and
838  `configure`).
839 
840 - Language definition syntax streamlined with [variants][] and
841  [beginKeywords][].
